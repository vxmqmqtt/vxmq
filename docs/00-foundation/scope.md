# 范围定义

本文档定义项目范围边界。阶段状态、完成度和下一步规划统一查看 [`../01-status/current-status.md`](../01-status/current-status.md) 与 [`../01-status/milestones.md`](../01-status/milestones.md)。

## 长期承诺范围

- 实现一个严格遵循 MQTT 协议的 Broker，而不是 MQTT Client 或应用侧 SDK。
- 同时支持 MQTT 3.1.1 与 MQTT 5，并在文档中明确版本差异。
- 建立可验证的核心能力：
  - 连接管理
  - 发布订阅主链路
  - 会话状态
  - QoS 语义
  - Retained Message
  - Will Message
  - MQTT 5 关键属性
  - 鉴权与基础可观测性
- 对关键行为提供自动化测试、阶段验收和决策记录。

## 当前不在短期范围内

- 多节点集群与分布式一致性。
- 管理控制台、复杂管理 API 或完整运维平台。
- 大规模插件系统。
- 提前为未知生产场景做过度抽象。

## 长期不承诺范围

- 在没有明确产品需求之前，把项目扩展成 IoT 平台、规则引擎或消息流处理平台。
- 为兼容个别客户端的非标准行为而破坏规范优先级。
- 在缺少验证和文档前提下，引入大范围“先实现再解释”的协议行为。

## 范围控制原则

- 先完成可验证的最小闭环，再扩展高级能力。
- 规范优先于实现便利，状态清晰优先于功能堆积。
- 新增范围必须明确其协议依据、架构落点、测试要求和阶段归属。
