# 项目愿景

`vxmq` 的目标不是做一个“可以收发消息的演示程序”，而是构建一个严格遵循 MQTT 协议、同时支持 MQTT 3.1.1 与 MQTT 5 的 Java Broker，并逐步演进为具备长期维护价值的工程化项目。

## 项目目标

- 以协议正确性为第一优先级，实现可验证的 MQTT Broker。
- 在 Quarkus + Mutiny + Vert.x 技术栈上建立清晰、可演进的单机 Broker 内核。
- 通过文档、ADR、自动化测试和阶段验收形成可持续开发基础。
- 在每个里程碑结束时，都能够明确回答“当前已经支持什么、尚未支持什么、下一阶段重点是什么”。

## 愿景边界

当前项目聚焦于 Broker 本体实现，不以“尽快堆满所有高级特性”为目标。短期优先级是先把协议主链路、状态语义和验证闭环做扎实，再逐步扩展高级能力、运维能力和生产化能力。

## 成功标准

- 协议行为能够持续对照 OASIS MQTT 3.1.1 与 MQTT 5 规范进行实现和验证。
- 文档能同时承担规范真相、状态总览和历史记录三种职责，但三者边界清晰，不相互混淆。
- 每个重要行为都有自动化测试或明确的验证计划，而不是依赖手工验证或记忆。
- 架构能够支撑后续引入会话状态、QoS 1/2、Retained Message、Will Message、鉴权和可观测性能力。

## 当前定位

项目当前已完成 `M1 最小闭环`，已具备单机 Broker 的最小连接、订阅、发布、取消订阅与断连主链路。当前状态与阶段结论统一记录在 [`../01-status/current-status.md`](../01-status/current-status.md)。
