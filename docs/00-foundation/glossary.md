# 术语表

本文档统一项目中的核心术语，避免设计文档、协议文档和代码注释对同一概念使用不同表述。

## 连接

客户端与 Broker 之间的一条网络连接。连接级状态通常包括：已打开、已认证接入、断连中、已关闭。

## 会话

与 `clientId` 绑定的逻辑状态集合。当前 `M1` 中会话主要承载订阅集合与当前绑定连接；后续里程碑将扩展为会话过期、未完成消息、离线状态等。

## Client Identifier

客户端标识。它是连接接管、会话定位和订阅归属的核心键。

## Topic Name

发布消息时使用的具体主题名。它不能包含 MQTT 通配符。

## Topic Filter

订阅时使用的主题过滤器。它允许使用 `+` 和 `#` 通配符，并按 MQTT 规范匹配 `Topic Name`。

## Subscription

客户端会话对某个 `Topic Filter` 的订阅关系。当前项目中订阅关系同时存在于会话视图和路由视图中。

## 路由

Broker 根据发布消息的 `Topic Name`，查找并确定应接收该消息的订阅集合的过程。

## 连接接管

当新的连接使用已被占用的 `clientId` 成功接入时，旧连接被替换并关闭的过程。MQTT 5 与 MQTT 3.1.1 在对旧连接的处理方式不同。

## Keep Alive

客户端在 CONNECT 中声明的最大空闲时间。当前项目依赖 `vertx-mqtt` 的内置机制处理超时探测与断连。

## 协议主链路

指 `CONNECT`、`SUBSCRIBE`、`UNSUBSCRIBE`、`PUBLISH`、`DISCONNECT` 等对 Broker 基本可用性最关键的报文处理路径。
