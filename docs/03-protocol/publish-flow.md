# PUBLISH 流程

本文档定义 `M1` 阶段发布链路的处理流程，覆盖：

- 入站 PUBLISH 校验。
- Topic Name 匹配与路由。
- QoS 0 投递。
- 出站消息下发。
- 失败路径与边界约束。

## 目标

`M1` 的发布流程要保证：

1. 已连接客户端可稳定发布 QoS 0 消息。
2. 消息可以按 Topic Name 正确路由到匹配订阅者。
3. 发布链路不提前耦合 `M2` 才需要的 QoS 1 / QoS 2 状态机。

## 输入与输出

输入：

- 已完成 CONNECT 的客户端发送 PUBLISH。

输出：

- 对 QoS 0 入站消息，不返回 ACK 报文。
- 将消息投递给所有匹配订阅者。

## 前置约束

- 连接状态必须为 `CONNECTED`。
- `M1` 只支持 QoS 0。
- `M1` 不处理 Retained Message 存储。
- `M1` 不处理离线消息堆积。

## 流程总览

```text
read PUBLISH
  -> decode and basic validation
  -> protocol validation
  -> validate topic name and qos
  -> resolve target subscriptions
  -> build outbound publish
  -> deliver to matched sessions
  -> record result
```

## 详细步骤

### 1. 接收与解码

- `codec` 将字节流解码为 PUBLISH 报文对象。
- 若报文结构非法，进入协议错误路径。

`M1` 关注点：

- 固定报头中的 QoS 位必须合法。
- QoS 0 下 DUP 标志不应影响语义判断。
- Topic Name 必须存在。

### 2. 协议语义校验

至少校验以下内容：

- 连接已处于 `CONNECTED`。
- 当前阶段仅接受 QoS 0 入站消息。
- Topic Name 合法且不是 Topic Filter。
- 保留位和不支持属性按当前阶段策略处理。

`M1` 建议：

- 对 QoS 1 / QoS 2 请求直接明确拒绝，不做半支持实现。
- 对 MQTT 5 高级属性，若未支持，应明确忽略范围或拒绝策略。

### 3. Topic Name 校验

校验原则：

- Topic Name 不为空。
- Topic Name 不能包含订阅通配符 `+` / `#`。
- 若后续引入系统保留主题前缀，应在这里统一处理。

## 路由解析

- `protocol` 调用 `routing` 查询匹配订阅者。
- `routing` 使用 Topic Filter 匹配规则返回目标会话集合。

`M1` 约束：

- 路由结果只包含当前在线会话。
- 不为离线会话缓存消息。
- 不考虑共享订阅分发。
- 同一客户端若有多条非共享订阅同时命中，`M1` 按 `ADR-0003` 去重后只投递一次。

## 出站消息构造

对每个目标订阅者：

1. 构造出站 PUBLISH 对象。
2. 根据订阅关系确定出站 QoS。
3. 通过统一发送出口写回目标连接。

`M1` 建议：

- 出站消息对象与入站消息对象分离，避免直接复用导致状态串扰。
- 即使当前只做 QoS 0，也为后续添加 Packet Identifier 保留扩展位置。

## 投递规则

### 目标连接在线

- 立即发送消息。

### 目标会话无在线连接

- `M1` 直接跳过，不缓存。

### 发布者同时是订阅者

- 若 Topic 匹配其自身订阅，允许收到回投消息。

原因：

- 这符合普通发布订阅语义。
- 不应隐式引入“禁止自投递”的非标准行为。

## 投递失败处理

### 单个目标写失败

- 记录错误。
- 将该目标连接进入断连清理流程。
- 不影响其他目标继续投递。

### 路由查询失败

- 记录错误。
- 当前发布处理终止。
- 按内部异常路径关闭当前连接或返回错误策略处理。

## 顺序与一致性

`M1` 要求：

- 同一连接入站报文按接收顺序处理。
- 对同一目标连接的出站消息保持发送顺序。
- 不为了并发投递破坏单连接内顺序保证。

## 可观测性

应至少记录以下信息：

- 入站 PUBLISH 总数。
- 投递命中订阅者数量。
- 投递失败数量。
- 因协议错误被拒绝的发布数量。

## 失败路径

### Topic Name 非法

- 拒绝处理或关闭连接。

### QoS 不支持

- 明确拒绝或关闭连接。

### 连接状态非法

- 关闭连接。

### 内部异常

- 记录日志。
- 进入统一错误处理路径。

## M1 验收点

- 已连接客户端可发布 QoS 0 消息到普通主题。
- 精确匹配订阅者可收到消息。
- `+` / `#` 通配订阅者可收到匹配消息。
- 未订阅客户端不会收到消息。
- 非法 Topic Name 被拒绝。
- QoS 1 / QoS 2 发布在 `M1` 被明确拒绝。
- 集成测试覆盖单订阅者、多订阅者、通配匹配、非法主题、非法 QoS 五类路径。

## 待确认项

- `M1` 是否接受 RETAIN 标志但忽略存储，还是直接拒绝。
- `M1` 对未支持 MQTT 5 发布属性的策略是忽略还是拒绝。
- 发布路径内部是串行投递还是按连接维度有限并发。
