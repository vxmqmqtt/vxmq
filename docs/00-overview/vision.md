# 项目愿景

## 项目目标

`vxmq` 的目标是实现一个基于 Java 的 MQTT Broker，在严格遵循 MQTT 协议的前提下，同时支持 MQTT 3.1.1 与 MQTT 5，并具备面向生产演进的工程基础。

项目应同时满足以下三类目标：

1. 协议正确性：严格遵循 MQTT 3.1.1 与 MQTT 5 规范，重点保证连接、会话、订阅、发布、QoS、保留消息、遗嘱消息、属性语义等核心行为正确。
2. 工程可演进性：具备清晰的模块边界、可测试性、可观测性与配置管理能力，为后续扩展集群、持久化、认证鉴权奠定基础。
3. 可验证性：每个重要特性都应有明确的验收方式，避免“功能看起来能跑”但语义不完整的实现。
4. 决策可追溯性：对协议未明确规定或存在实现自由度的行为，在实现前先核对规范、参考同类产品官方资料或实测行为，并记录为项目决策。

## 愿景边界

本项目不是一个只验证技术选型的 Demo，也不是只支持单一协议版本的轻量消息中转器。我们要建设的是一个可持续迭代的 MQTT 3.1.1 / MQTT 5 Broker 核心。

## 成功标准

以下条件可视为阶段性成功：

- MQTT 5 特性矩阵中的核心项进入“已实现 + 已验证”状态。
- 核心模块职责明确，主要交互流程有文档可追溯。
- 具备系统化测试，包括协议语义验证、互操作性验证和性能基线验证。
- 新成员可通过 `README.md` 和 `docs/` 快速建立项目认知。

## 当前假设

- 运行时基础框架为 Quarkus。
- 实现语言为 Java。
- 初期优先单机 Broker 核心能力，集群与高级运维能力在后续里程碑推进。
