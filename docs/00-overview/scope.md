# 范围定义

## 当前范围

当前已完成的 `M1` 目标是形成一个可运行、可测试、具备 MQTT 3.1.1 / MQTT 5 最小闭环语义的单机 Broker。

`M1` 已纳入范围的能力：

- 客户端连接与断连。
- MQTT 3.1.1 / MQTT 5 CONNECT / CONNACK 基础能力。
- 发布与订阅基本链路。
- QoS 0。
- 主题过滤与通配符订阅。
- 基础配置与日志。

`M2` 及后续阶段计划承接的能力：

- QoS 1 / QoS 2。
- 会话状态管理与过期控制。
- 保留消息。
- 遗嘱消息。
- 健康检查与指标。
- TLS 与用户名密码鉴权。

## 暂不承诺范围

以下内容在当前规划中默认不承诺近期完成，除非后续明确提升优先级：

- 集群与多节点路由。
- 跨节点会话迁移。
- 多租户能力。
- 完整控制台或复杂管理 UI。
- 大规模持久化优化。
- 高级安全功能，例如细粒度策略引擎或企业级 IAM 集成。

## 范围控制原则

- 优先保证 MQTT 协议语义正确，再考虑性能优化。
- 优先形成最小闭环，再逐步扩展能力面。
- 不为了“覆盖所有高级功能”破坏核心模块的可维护性。
- 对协议未明确规定的行为，先核对规范，再参考同类产品官方资料或实测结果，确认后再编码。

## 待确认事项

- `M2` 是否要求消息持久化落盘。
- `M3` 是否要求 TLS / 用户名密码鉴权同时到位。
- 性能目标的具体数值与验证基线。
