# ADR-0001 协议兼容性与决策原则

- 状态：已接受
- 日期：2026-03-14

## 背景

项目早期文档主要围绕 MQTT 5 展开，但项目实际目标已经明确为：

- 严格遵循 MQTT 协议。
- 同时支持 MQTT 3.1.1 与 MQTT 5。

同时，MQTT 规范中存在两类需要额外处理的场景：

1. 不同协议版本对同一行为存在差异。
2. 规范存在 `MAY`、实现可选或未规定细节的地方。

如果不先建立统一的决策原则，后续实现会出现：

- 文档和代码只覆盖单一版本。
- 相同问题在不同模块中被重复决策。
- 协议自由度被随意处理，导致互操作性不可预测。

## 决策

1. `vxmq` 的目标协议版本为 MQTT 3.1.1 与 MQTT 5。
2. 所有核心协议链路都必须显式考虑两个版本的差异。
3. 协议行为的判断优先级固定为：
   - OASIS MQTT 规范
   - 已接受 ADR
   - HiveMQ / EMQX / Mosquitto 的官方资料或实测行为
   - 新增项目 ADR
4. 对协议未规定、允许多种实现或包含 `MAY` 的行为，在实现前必须先查规范，再查同类产品资料或实测结论，最后形成项目决策。
5. 不允许为了缩短实现路径而静默忽略协议可见差异。

## 结果

- 文档、设计、实现将以“双版本兼容”为默认前提。
- 版本差异会作为显式设计问题处理，而不是散落在局部实现中。
- 协议自由度问题进入“先确认、再实现”的节奏。

## 影响

- 早期设计文档中仅面向 MQTT 5 的表述需要逐步修正。
- 测试计划必须覆盖 MQTT 3.1.1 与 MQTT 5。
- 后续重要协议决策需要继续写入 `decisions/`。

## 参考依据

- OASIS MQTT 3.1.1 规范
- OASIS MQTT 5 规范
- HiveMQ / EMQX / Mosquitto 官方文档与后续互操作性验证
